This migration creates the subjects table for the Meskerem Primary e-School system. It includes comprehensive Row Level Security (RLS) policies that control access based on user roles. The table stores subject information with grade levels and department associations.

SQL Migration Script

migration_subjects.sql 
BEGIN;

-- Create subjects table
CREATE TABLE IF NOT EXISTS public.subjects (
  id BIGSERIAL PRIMARY KEY,
  name TEXT NOT NULL,
  grade_level TEXT,
  department TEXT,
  description TEXT,
  created_at TIMESTAMPTZ DEFAULT NOW()
);

ALTER TABLE public.subjects ENABLE ROW LEVEL SECURITY;

-- (Re)create policies cleanly
DROP POLICY IF EXISTS "Anyone can view subjects" ON public.subjects;
DROP POLICY IF EXISTS "Authenticated users can add subjects" ON public.subjects;
DROP POLICY IF EXISTS "Admins can update subjects" ON public.subjects;
DROP POLICY IF EXISTS "Admins can delete subjects" ON public.subjects;

-- Allow anyone to read
CREATE POLICY "Anyone can view subjects" 
  ON public.subjects
  FOR SELECT
  TO PUBLIC
  USING (true);

-- Allow authenticated users to insert
CREATE POLICY "Authenticated users can add subjects"
  ON public.subjects
  FOR INSERT
  TO authenticated
  WITH CHECK ((SELECT auth.uid()) IS NOT NULL);

-- Allow admins to update
CREATE POLICY "Admins can update subjects"
  ON public.subjects
  FOR UPDATE
  TO authenticated
  USING (
    EXISTS (
      SELECT 1
      FROM public.teachers t
      WHERE t.id = (SELECT auth.uid())
        AND t.is_admin = true
    )
  )
  WITH CHECK (
    EXISTS (
      SELECT 1
      FROM public.teachers t
      WHERE t.id = (SELECT auth.uid())
        AND t.is_admin = true
    )
  );

-- Allow admins to delete
CREATE POLICY "Admins can delete subjects"
  ON public.subjects
  FOR DELETE
  TO authenticated
  USING (
    EXISTS (
      SELECT 1
      FROM public.teachers t
      WHERE t.id = (SELECT auth.uid())
        AND t.is_admin = true
    )
  );

COMMIT;

Prerequisites & Dependencies

Required Table: This migration depends on the public.teachers table existing with the following structure:

public.teachers
  • id UUID PRIMARY KEY (should match auth.uid())
  • is_admin BOOLEAN DEFAULT false
  • email TEXT
  • full_name TEXT

Note: If the teachers table doesn't exist yet, create it first or modify the policy conditions accordingly.

Testing the Policies

After applying the migration, test the RLS policies with these queries:

1. Public read (should work without auth)

SELECT * FROM public.subjects;

2. Insert as authenticated user

INSERT INTO public.subjects (name, grade_level, department) 
VALUES ('Mathematics', '5', 'STEM');

3. Update as admin (requires is_admin = true)

UPDATE public.subjects 
SET description = 'Advanced mathematics curriculum' 
WHERE name = 'Mathematics';
